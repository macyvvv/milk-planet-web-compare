# shopページ修復WBS

## 目的
branch1/2/3配下の shop/*/index.html に発生していた表示崩れを、リンク切れ・CSS影響・相対パス不整合の観点から整理し、原本に近い表示になるように修正する。

## 対象範囲
- proposal/branch1/shop/*/index.html
- proposal/branch2/shop/*/index.html
- proposal/branch3/shop/*/index.html

## 作業分解
1. 現状把握
   - 各ページで読み込まれている CSS を確認する
   - リンク切れの有無を確認する
   - 相対パスの誤りを洗い出す
2. 直接修正
   - 予定外の proposal.css 影響を排除する
   - メニューリンクを同一ディレクトリ配下の実在パスに直す
   - 必要に応じて相対パスを統一する
3. 検証
   - 参照先の存在確認を実施する
   - 追加CSSの読み込みが残っていないか確認する
   - 主要なショップページで差分が減っていることを確認する
4. 自己批判と再確認
   - 初回の確認ではリンク切れだけを見ていたことを反省し、CSS側の影響も含めて再評価する
   - 修正後に「他に影響しそうなパスや読み込み」がないか再点検する

## 実施内容
- メニューリンクを `menu/` へ統一
- 各ショップページから `proposal.css` の読み込みを除去
- 参照先存在確認と再確認を実施

## 自己批判メモ
- まずリンク切れだけを見ていたため、見出し・ヘッダー周りに強い影響を与える `proposal.css` の読み込みを見落としやすかった。
- 今回の修正では、リンク切れと CSS 影響を分けて扱うのではなく、同時に直す方針に切り替える。
- 変更は広範囲に及ぶため、対象を一括で修正し、検証まで含めて完了させる。

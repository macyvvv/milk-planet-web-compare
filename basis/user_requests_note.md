# User Requests Note

## 目的
過去の会話でユーザーが明示した要望を、後から参照できるように整理する。
これは設計判断のメモであり、実装仕様の最終確定ではない。

## 主要要望

### 1. 現状再現
- `currently/` を正本として固定したい。
- `proposal/branch1/` で現状をできるだけ完全に再現したい。
- JS で動くカルーセルや各種挙動も含めて再現したい。

### 2. 比較環境
- `currently` と `proposal` を実ブラウザで比較できるようにしたい。
- GitHub Pages にデプロイしたい。

### 3. 改修方針
- 改修版は、同じ画像や HTML を使い回すレスポンシブ範囲で収めたい。
- 更新の手間は増やしたくない。
- SNS の勢いが強い業界なので、店舗 Web サイトを持つ意味を定義したい。

### 4. ページ構造
- URL の数とページ責務を整理したい。
- ページ内遷移や iframe 的なものは SPA に寄せる選択肢を検討したい。
- ただし、まずは現状を理解したい。

### 5. 共通課題
- トップだけでなく、全ページ共通でヘッダー / ナビの扱いを見直したい。
- クリック可能要素は見た目で分かるようにしたい。
- 店舗ページ共通UIも揃えたい。

### 6. トップページ
- トップのカルーセルを小さくしたい。
- 画像の上下は切りたくない。
- カルーセルの左右に前後の画像を薄く見せたい。
- ドット表示など、現在位置が分かる表現が欲しい。

### 7. ハンバーガーメニュー
- 画面最上部のメニューはハンバーガー化したい。
- ただし `mobile-head` のような常設領域ではなく、ボタン近くにコンテキスト的に出したい。
- ボタンは「めにゅう」として意味が伝わる方が良い。

### 8. `しすてむ＆めにゅう`
- `div#sys` の中で、見出しと各店アイコンを役割分離したい。
- 上半分に見出し、下半分に各店アイコンを置きたい。
- 各店アイコンはそれぞれのシステム表へリンクしたい。

### 9. 店舗一覧ページ
- 店舗名テキストを各店のバナーに差し替えたい。
- バナーと説明文の距離感は元デザインを尊重したい。
- 余白や配置は元のイメージを保ちたい。

### 10. 個別店舗ページ
- `店舗名` / `店舗コンセプト` / `店内写真` / `アクセス` のセット構造を維持したい。
- このうち店舗名部分を、アクセス領域内のバナーに差し替えたい。
- 1スタックの中にバナーが2個ある状態にしたい。

### 11. キャスト
- キャスト写真を HTML に組み込みたい。
- フィルター的な JS 挙動を保ちたい。
- 配置や初期表示のズレをなくしたい。

### 12. デザインの優先
- 指示していない部分は基本的に current と同じ仕様にしたい。
- ただし proposal 側は目標達成のために新規 CSS / JS を使ってよい。

### 13. カルーセルの最終意図
- 表示幅はブラウザ全体を基準にしたい。
- 中央の 1 枚と、その前後 1 枚ずつ、計 3 枚を見せたい。
- `1:8:1` は表示領域の比率として扱いたい。
- 画像サイズそのものの比率は変えたくない。
- 左右は大半が見切れてよい。

## 補足
- このメモは、以後の proposal 改修での参照用。
- 仕様として確定したものではなく、要望の記録として扱う。

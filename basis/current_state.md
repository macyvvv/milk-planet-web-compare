# Current State

## 再現対象
- トップページ
- 店舗一覧
- キャスト一覧
- 採用ページ
- 各店舗のメニュー詳細

## 現在版の特徴
- SP前提の縦スクロール構成が強い。
- PC時は余白や列構成の再設計が不足している。
- CTA の見た目が弱く、リンクと装飾の境界が曖昧。
- トップのカルーセルは主に印象づけ用途で、比較・選択には寄与していない。
- 店舗一覧と個別店舗詳細は、情報の出し方がページごとにばらつく。

## 現時点の正本方針
- `currently/` は現状再現の正本として固定する。
- `proposal/branch1/` は改修案の比較用として扱う。
- トップページのメニュー、トップカルーセル、`しすてむ&めにゅう`、キャスト一覧は別論点として分離して扱う。
- `しすてむ&めにゅう` はページ内アンカーとして残しつつ、提案版では見せ方と責務を再設計する。

## 取得済み素材
- トップのメインビジュアル
- 店舗紹介用の写真、ロゴ、地図
- キャスト一覧関連の画像
- 採用バナー
- 各店舗のメニュー画像

## まだ注意が必要な点
- 一部の外部依存はローカル完結していない可能性がある。
- 動的に差し込まれる Instagram / TikTok 系は、完全な静的再現に追加作業が必要。
- `eventlist.js` や子ページの JS に依存する領域は、初回の見た目再現と挙動再現を分けて扱うべき。

## ここからの使い方
- `currently/` を現在版の参照実装として固定する。
- その上で、PC最適化、CTA強化、カルーセル縮小または代替を比較案として作る。
- `proposal/branch1/` では、トップメニューとトップカルーセルの整理を優先し、`しすてむ&めにゅう` は別セクションとして扱う。
